# Stocket 前端设计风格规范

- 日期：2026-07-12
- 状态：已确认，等待书面规范复核
- 视觉方向：薄荷奶油（Mint Cream）
- 产品气质：温暖轻松、明亮现代、清晰可信

## 1. 设计定位

Stocket 是家庭生活助手，而不是企业仓储后台。界面应让家庭成员感到轻松、安心和可掌控，同时准确呈现数量、日期、位置和风险等库存信息。

设计关键词为：**温暖、轻盈、清楚、可靠**。

不采用以下方向：

- 不使用冷峻的企业蓝、大面积深色或密集表格塑造专业感；
- 不依赖拟物纹理、手写字体或卡通插画制造家庭感；
- 不让装饰、渐变和动效盖过库存状态与操作结果；
- 不把所有内容放进带阴影的卡片，避免界面碎片化。

## 2. 核心原则

### 2.1 家庭感来自语气和呼吸感

使用奶油色页面背景、薄荷色功能面、自然的中文文案和适度圆角。移动端保持舒展，主任务有充分留白。家庭感不以降低信息清晰度为代价。

### 2.2 一个页面只有一个主操作

每个页面或步骤只保留一个高强调按钮。次要操作使用描边或文字按钮，危险操作使用独立的危险色，并在执行前说明影响。

### 2.3 状态不能只靠颜色表达

临期、过期、低库存、成功和失败必须同时使用文字、图标或形状。颜色只负责加强层级，不作为唯一信息来源。

### 2.4 手机舒展，桌面高效

移动端优先使用卡片、分组列表和分步表单，保证触控与快速扫读。桌面端沿用同一令牌和组件语义，缩小纵向间距，并在批量管理场景切换为表格和双栏布局。

### 2.5 动效解释变化

动效用于页面进入、列表更新、步骤切换和操作反馈，默认时长为 160–240ms。禁止无目的循环动画和大幅弹跳，并支持 `prefers-reduced-motion`。

## 3. 色彩系统

### 3.1 品牌与中性色

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `--st-color-bg` | `#FBF8EF` | 页面奶油底色 |
| `--st-color-surface` | `#FFFEF9` | 卡片、输入框、导航面 |
| `--st-color-primary-soft` | `#DFF4EC` | 快捷操作背景、选中项、柔和提示 |
| `--st-color-primary` | `#167A63` | 主按钮、活动导航、链接、焦点边框 |
| `--st-color-primary-hover` | `#126B56` | 主操作悬停态 |
| `--st-color-primary-active` | `#0F5D4B` | 主操作按下态 |
| `--st-color-text` | `#20332C` | 标题和正文主色 |
| `--st-color-text-muted` | `#6C7B75` | 次要说明和元数据 |
| `--st-color-border` | `#E4E8DF` | 分隔线和默认边框 |

### 3.2 语义色

| 令牌 | 色值 | 用途 |
| --- | --- | --- |
| `--st-color-success` | `#167A63` | 成功、健康、已完成 |
| `--st-color-warning` | `#E5A33D` | 临期、需注意、可继续操作的风险 |
| `--st-color-danger` | `#D65353` | 过期、失败、不可逆操作 |
| `--st-color-accent` | `#EF765F` | 少量高优先提醒，不用于常规主按钮 |
| `--st-color-info` | `#3D7F8B` | 中性系统消息和帮助信息 |

语义色必须配置浅色背景和深色文本变体，并满足 WCAG AA 对比度。正文不得直接使用浅珊瑚色或浅琥珀色。

## 4. 字体与排版

中文优先使用系统字体栈，避免额外加载覆盖不完整的中文 Web Font：

```css
font-family: Inter, ui-sans-serif, system-ui, -apple-system,
  BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
```

- 展示标题：28–36px，字重 700–760，紧凑行高与轻微负字距；
- 页面标题：24–28px，字重 700；
- 区块标题：18–20px，字重 650–700；
- 正文：14–16px，行高 1.55–1.7；
- 元数据与辅助说明：12–13px，不小于 12px；
- 数量使用等宽数字特性 `font-variant-numeric: tabular-nums`；
- 大段文案保持左对齐，不使用居中排版。

首页可以使用自然问候语；表单、错误和确认信息应直接、具体，避免模糊的拟人化表达。

## 5. 空间、圆角与层次

- 基础空间单位为 4px，常用间距为 8、12、16、24、32px；
- 移动页面水平边距为 16–20px，桌面内容区为 24–32px；
- 控件圆角为 12px，普通卡片为 16px，重点卡片为 20px，浮层为 20–24px；
- 胶囊圆角只用于短标签、状态和筛选项，不用于所有按钮；
- 默认层次由背景色、1px 边框和间距建立；
- 阴影只用于对话框、浮层、悬浮主操作和需要明确抬升的重点区域；
- 桌面主内容最大宽度建议为 1440px，阅读型内容最大宽度为 720px。

## 6. 图标与图像

- 采用圆润但简洁的线性图标，默认线宽和视觉尺寸保持一致；
- 图标用于辅助识别，关键操作仍保留文字标签；
- 商品图片使用圆角矩形并统一裁切，缺图时用低饱和分类图标占位；
- 仅在首次初始化、真正空状态和任务完成等低频场景使用插画；
- 插画应采用扁平几何形、薄荷绿和少量珊瑚色，不使用高细节 3D 素材或随机风格 emoji 作为正式图标。

## 7. 核心组件语言

### 7.1 按钮

- 主按钮使用行动绿实底，文字为白色；
- 次按钮使用浅色面或描边；三级操作使用文字按钮；
- 危险按钮使用危险色，不能以品牌绿确认删除、报废或撤销；
- 按钮文案使用明确动词，如“完成入库”“转移 2 盒”“撤销邀请”；
- 移动端主要按钮高度不低于 48px，其他可点击区域不低于 44×44px。

### 7.2 表单

- 标签始终可见，不以 placeholder 代替字段名；
- 输入框默认高度为 44–48px，焦点态使用 2px 行动绿边框和柔和外圈；
- 校验信息紧邻字段，说明错误原因和修复方式；
- 入库采用四步向导，一屏聚焦一个判断，自动保存草稿；
- 日期、数量和单位应显示业务语境，提交前明确展示库存变化。

### 7.3 列表与表格

- 移动物品列表每行优先展示名称、位置、数量和最紧急状态；
- 次要元数据保持一到两行，不能依赖进入详情才能看见核心风险；
- 桌面端批量管理使用表格，表头固定，数量右对齐，状态列使用文字标签；
- 行操作优先收进上下文菜单，仅保留最高频操作；
- 筛选项使用紧凑的浅色筛选按钮，活动项使用薄荷面与深绿文字。

### 7.4 导航

- 移动端保持：首页、物品、中央快捷入库、提醒、我的；
- 中央入库使用高强调绿色按钮，但不通过持续动画吸引注意；
- 桌面端使用左侧导航，主区域顶部提供搜索、页面操作和账户入口；
- 当前导航同时使用颜色、图标状态和文字字重表示。

### 7.5 反馈与浮层

- 普通成功使用轻量 Toast，不阻断后续操作；
- 失败信息说明发生了什么、数据是否保留以及下一步；
- 删除、报废、退出全部会话等高风险操作使用对话框，并显示具体对象与后果；
- 底部操作面板用于移动端短选项，对话框用于明确确认，抽屉用于保留上下文的详情编辑；
- 加载使用贴近真实布局的骨架屏，按钮提交态防止重复操作。

## 8. 状态设计

### 8.1 库存与提醒

- 正常：绿色或中性样式，避免整个列表都是绿色标签；
- 临期：琥珀色，显示具体剩余天数或日期；
- 过期：危险色，明确写“已过期”，并提供处理入口；
- 低库存：珊瑚或危险色，根据严重程度显示剩余数量；
- 已归档、已报废和已撤销：使用低饱和中性色，不与当前有效数据混淆。

### 8.2 空、加载与错误

- 空状态解释为什么为空，并提供一个最相关的下一步；
- 首次加载用骨架屏，局部刷新不清空已有内容；
- 网络失败保留可用内容和表单草稿，提供重试；
- `401` 回到登录状态，`403` 解释权限原因，`409` 展示冲突对象和安全恢复方式；
- 离线时明确区分“可查看缓存”和“写操作需要联网”。

## 9. 响应式策略

### 9.1 移动端

- 单列布局，卡片间距 12–16px；
- 首页按搜索、快捷入库、风险摘要、待处理事项排列；
- 详情页优先呈现当前状态与高频操作，历史和附件按区块展开；
- 表单底部主操作可粘滞，但不得遮挡校验信息和系统安全区。

### 9.2 平板与桌面端

- 768px 起允许双栏；1024px 起启用固定侧边导航；
- 列表转为密度适中的表格，行高建议 48–56px；
- 批量整理采用“表格 + 右侧详情抽屉”，减少页面往返；
- 管理配置页面使用左侧二级导航和右侧内容，不复刻移动底栏；
- 桌面端增加悬停反馈，但所有功能仍可用键盘完成。

## 10. 无障碍与内容规范

- 正文和控件满足 WCAG 2.2 AA 对比度；
- 焦点环必须清晰，不得只依赖浏览器默认样式或直接移除 outline；
- 支持键盘顺序、跳过导航、表单错误汇总和屏幕阅读器状态播报；
- 状态标签包含可读文字，图标装饰正确隐藏；
- 日期优先显示绝对日期，必要时同时补充“明天”“还有 3 天”；
- 数量始终带单位；位置显示完整路径或提供可访问的路径说明；
- 成功文案简短，错误文案不责备用户，危险确认文案必须具体。

## 11. 与 Element Plus 的关系

继续使用 Element Plus 承担底层交互与可访问性能力，但通过 Stocket 语义令牌统一主题。业务页面不直接散落 Element Plus 默认色值，也不以全局深度选择器堆叠临时覆盖。

建议建立以下层级：

1. `tokens.css`：颜色、排版、间距、圆角、阴影和动效；
2. `element-theme.css`：把 Element Plus CSS 变量映射到 Stocket 令牌；
3. 基础业务组件：`StPageHeader`、`StStatusTag`、`StEmptyState`、`StItemRow`、`StFormActions`；
4. 页面组件只组合上述语义组件，不重新定义局部品牌风格。

## 12. 验收标准

- 首页、物品列表、入库向导、提醒列表、登录和管理员页面看起来属于同一产品；
- 在 320px 宽手机上不发生水平溢出，主要触控区不小于 44px；
- 桌面表格能够提高信息密度，但不缩小到 12px 以下的正文；
- 正常、临期、过期、低库存、离线和失败状态均不只依赖颜色；
- 键盘可完成登录、搜索、入库向导、筛选和对话框操作；
- `prefers-reduced-motion` 下不执行非必要转场；
- 设计令牌覆盖品牌色、文字、边框、状态、间距、圆角、阴影和动效；
- 新页面优先复用语义组件，避免新增不受控的色值和圆角。

## 13. 后续实施范围

本规范只确定前端设计风格和设计系统边界，不在本阶段重做业务页面。后续实施计划应先完成设计令牌、Element Plus 主题映射和基础组件，再按身份流程、移动应用外壳、物品与库存页面的顺序落地。
