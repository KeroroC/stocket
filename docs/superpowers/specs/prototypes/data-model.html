核心数据模型：档案与库存分离

一个物品档案可以对应多个库存持有单元；批次和独立资产共享统一的移动流水

档案与配置 当前库存 不可变流水 支撑能力

分类 Category

  • 树形层级
  • 分类属性模板 JSONB
  • 批次/资产默认类型

物品档案 Item Definition

  • 名称、品牌、型号、规格、单位
  • 条码、标签、默认保质期
  • 核心字段 + 经模板校验的 JSONB 属性
一个档案→多个库存单元

库存单元 Inventory Entry

  • 类型：BATCH / ASSET
  • 当前位置、可用数量、入库时间
  • 生产日期、过期日期、状态、版本号
批次详情
批次号、数量可大于 1、保质期
资产详情
资产号、序列号、保修期、数量恒为 1

位置 Location

  • 房屋 → 房间 → 柜子 → 层板
  • 无限层级，自引用父节点
  • 可生成二维码用于扫码定位
每次变化→追加流水

库存流水 Inventory Movement

  • IN / OUT / TRANSFER / ADJUST / DISCARD
  • 变动数量、变动前后快照
  • 来源位置、目标位置、操作人、原因
  • 幂等键，防止重复提交

Outbox / Reminder / Audit

  • 事务内写入业务事件
  • 异步生成临期提醒与通知任务
  • 保留关键配置及权限操作审计
流水是历史事实流水只追加、不修改；错误通过一条反向调整流水纠正。
库存快照用于快速查询Inventory Entry 保存当前数量，但只能由库存服务在同一数据库事务中更新。
并发不允许出现负库存变动时锁定库存单元，校验余额，写流水并更新快照；版本号用于冲突检测。
已发生流水的档案不硬删除使用归档状态保留历史关联；分类、位置被引用时也不能直接删除。
日期类型语义明确生产日、过期日使用 date；入库、操作、通知时间使用带时区时间戳。
支持小数数量数量使用高精度数值，可表达 2 件、1.5 千克、0.75 升等库存。
