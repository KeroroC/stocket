# Stocket 桌面后台布局优化设计规格

- 日期：2026-07-15
- 状态：待用户复核书面规格
- 关联文档：
  - `docs/superpowers/specs/2026-07-10-stocket-design.md`
  - `docs/superpowers/specs/2026-07-12-frontend-design-style.md`
  - `docs/superpowers/plans/2026-07-12-mobile-pwa-workflows.md`
- 视觉方向：继续薄荷奶油（Mint Cream），不引入深色主题
- 实现路径：增强现有壳 + 桌面断点样式（方案 A）

## 1. 背景与目标

Stocket 是手机优先的家庭库存 PWA。现有桌面断点（≥1024px）已有左侧栏并隐藏底栏，但整体仍像“加宽的移动壳”：

- 侧栏菜单扁平，业务入口与管理入口混排；
- 缺少后台常见的顶栏（搜索 / 快捷操作 / 账户）；
- 内容区信息密度偏低，表格化与工具条布局不完整。

本规格目标：在**不破坏移动端体验**的前提下，使电脑端呈现标准后端管理系统结构——左侧分组导航、顶部工具区、主内容区表格/双栏管理布局。

### 1.1 成功标准

- 视口 ≥1024px 时，已认证用户看到固定左侧分组侧栏 + sticky 顶栏 + 主内容，不再显示移动底栏。
- 视口 <1024px 时，布局、底栏、交互与当前实现一致（允许因共享组件产生的无感样式复用，但不得改变信息架构与主路径）。
- 桌面侧栏按业务分组；ADMIN 可见系统管理组；VIEWER 不显示入库。
- 首页、物品、入库、库存、提醒、个人及全部管理页在桌面下具备后台化布局（页头 + 工具条 + 表格/双栏）。
- 继续使用现有 mint cream 设计令牌与 `StPageHeader` / `StEmptyState` / `StFormActions` 等组件语言。
- 现有相关单测与 Playwright 响应式用例保持通过，并按需补充桌面壳与分组导航断言。

### 1.2 非目标

- 不修改后端 API、权限模型或业务规则；
- 不重做移动底栏与移动任务流；
- 不引入新的 UI 框架，不以 Element Plus Layout/Menu/Table 整体替换现有体系；
- 不实现深色侧栏主题、可折叠侧栏、多标签页、自定义表格列配置；
- 不拆分为独立桌面应用或第二套路由树。

## 2. 决策摘要

| 决策项 | 选择 |
| --- | --- |
| 改动范围 | 全面桌面后台化（外壳 + 全部业务/管理页） |
| 视觉气质 | 保留薄荷奶油，只调整结构与密度 |
| 移动端 | 保持现状 |
| 侧栏组织 | 按业务分组：概览 / 库存业务 / 系统管理 / 个人 |
| 实现路径 | 方案 A：增强 `PwaAppShell` / `DesktopSidebar` + 桌面断点样式 |

## 3. 断点与外壳结构

### 3.1 断点

| 范围 | 外壳 |
| --- | --- |
| `< 1024px` | 移动壳：`MobileTabBar` + 内容区底栏安全区 padding |
| `≥ 1024px` | 后台壳：`DesktopSidebar` + `DesktopTopBar` + 主内容；隐藏底栏 |

断点与现有实现保持一致，避免额外分裂测试矩阵。

### 3.2 桌面布局示意

```text
┌────────────┬──────────────────────────────────────┐
│  Brand     │  顶栏：全局搜索 | 快捷入库 | 账户菜单 │
│            ├──────────────────────────────────────┤
│  分组侧栏  │                                      │
│  · 概览    │           主内容区 (#main-content)    │
│  · 业务    │           max-width ≈ 1440px          │
│  · 管理*   │           内边距 24–32px              │
│  · 个人    │                                      │
└────────────┴──────────────────────────────────────┘
* 系统管理组仅 ADMIN 可见
```

### 3.3 组件职责

| 组件 | 职责 |
| --- | --- |
| `PwaAppShell` | 组装已认证应用壳；桌面挂载侧栏/顶栏/主区；移动挂载底栏；无路由时继续回退 `AppShell`（legacy） |
| `DesktopSidebar` | 品牌区 + 分组导航；当前路由 `aria-current="page"` |
| `DesktopTopBar`（新增） | 全局搜索、快捷入库、账户菜单（资料 / 通知设置 / 退出） |
| `MobileTabBar` | 保持五个入口：首页 / 物品 / 入库 / 提醒 / 我的 |
| 主内容 `RouterView` | 继续向页面透传 `account` / `role` 与 logout 事件 |

### 3.4 顶栏行为

- **全局搜索**：复用现有 `GlobalSearch` 能力；桌面优先置于顶栏。
- **快捷入库**：链接至 `/receive`；ADMIN / MEMBER 可见；VIEWER 隐藏。
- **账户菜单**：展示 `displayName` 与角色；菜单项包含「我的账户」「通知设置」「退出登录」。
- 侧栏不再强调底部大号退出按钮；退出统一从账户菜单触发（若需兼容旧测试，可保留视觉弱化的次要退出入口，但不作为主路径）。

### 3.5 可访问性

- 提供 skip link，跳转目标 `#main-content`。
- 侧栏导航 `aria-label` 保持「桌面主导航」；分组标题使用非交互标题元素。
- 当前页同时通过颜色、字重与 `aria-current` 表达，不只依赖颜色。
- 可点击目标在桌面仍尽量满足既有触达尺寸约定，避免回归 E2E 最小高度检查时大面积失败；若个别紧凑表格操作需要例外，应在实现计划中单列并同步调整测试期望。

## 4. 侧栏分组与路由映射

### 4.1 分组

| 分组 | 入口 | 路由 | 可见性 |
| --- | --- | --- | --- |
| 概览 | 首页 | `/` | 全部已认证角色 |
| 库存业务 | 物品目录 | `/items` | 全部 |
| | 入库 | `/receive` | ADMIN、MEMBER |
| | 库存台账 | `/inventory` | 全部 |
| | 提醒中心 | `/reminders` | 全部 |
| 系统管理 | 成员管理 | `/admin/members` | ADMIN |
| | 邀请管理 | `/admin/invites` | ADMIN |
| | 分类管理 | `/admin/categories` | ADMIN |
| | 位置管理 | `/admin/locations` | ADMIN |
| | 通知失败 | `/admin/delivery-failures` | ADMIN |
| | 审计日志 | `/admin/audit-logs` | ADMIN |
| | 系统诊断 | `/admin/diagnostics` | ADMIN |
| 个人 | 我的账户 | `/profile` | 全部 |
| | 通知设置 | `/notification-settings` | 全部 |

### 4.2 规则

- 分组标题只作标签，不可点击。
- 若某分组在当前角色下无任何可见入口，则整组不渲染。
- 路由表与权限 meta 保持现状；前端隐藏不等于安全边界。
- 移动端 Profile 内的管理入口链接保留，方便小屏到达管理页。
- 图标继续使用 `@element-plus/icons-vue`；桌面侧栏「入库」为普通导航项，高强调仅保留移动中央按钮与顶栏快捷入口。

## 5. 各页面桌面形态

移动布局保持不变；下列要求仅针对 `≥1024px`。

### 5.1 通用骨架

1. `StPageHeader`：标题、可选说明、右侧主操作；
2. 工具条：搜索、筛选、次要操作，优先单行；
3. 主区：表格，或左树右内容；
4. 空态/错误：`StEmptyState` 与现有反馈样式。

### 5.2 首页 `/`

- 问候与概览标题保留产品语气；
- 顶栏已承载全局搜索时，首页大搜索可降级为次要或精简，避免双重主搜索竞争（实现时优先「顶栏主搜索 + 首页精简」）；
- 临期/过期/低库存/待处理以统计卡片横排展示；
- 「快捷入库」在首页改为工具级主按钮，不再使用大块移动 hero 作为桌面主视觉；
- 关注列表使用紧凑双列卡片或表格化列表。

### 5.3 物品 `/items`

- 左：分类/位置浏览树（可切换）；
- 右上：搜索 +「创建物品」；
- 右下：结果表格（名称、分类、关键状态等）；
- 详情/编辑优先右侧抽屉或居中对话框，减少整页切换打断浏览。

### 5.4 入库 `/receive`

- 向导内容居中，最大宽度约 40–48rem；
- 步骤条完整水平展示；
- 仍为四步向导，不改成单页大表单。

### 5.5 库存台账 `/inventory`

- 筛选条 + 结果表格；
- 消耗/调拨/调整在桌面使用居中对话框，替代底部 sheet 的移动观感。

### 5.6 提醒 `/reminders`

- 筛选 + 表格或分组紧凑列表；
- 状态、到期日、关联物品等列对齐，增强扫读效率。

### 5.7 管理页

覆盖：成员、邀请、分类、位置、通知失败、审计日志、系统诊断。

- 统一：页头 + 工具条 + 表格（或左树右编辑）；
- 分类/位置：强化左树右表单的桌面双栏对齐与宽度；
- 审计/失败：筛选表单 + 表格 + 既有分页/游标区域。

### 5.8 个人与通知设置

- 表单内容最大宽度约 36–40rem，左对齐；
- 降低“窄居中卡片”的移动感，但不引入复杂多栏设置导航。

## 6. 视觉与设计令牌

- 继续使用 `frontend/src/styles/tokens.css` 中的 mint cream 令牌；
- 侧栏：`--st-color-surface` 背景 + 右边框；宽度约 15–16rem；
- 顶栏：浅色、sticky、底部分割线；
- 当前导航：`--st-color-primary-soft` 底 + `--st-color-primary` 文字与字重；
- 主内容最大宽度建议 1440px，水平内边距 24–32px；
- 不把桌面改成冷灰企业后台或深色侧栏。

## 7. 实现约束与兼容

### 7.1 代码落点（预期）

- `frontend/src/components/PwaAppShell.vue`
- `frontend/src/components/navigation/DesktopSidebar.vue`
- 新增 `frontend/src/components/navigation/DesktopTopBar.vue`（名称可在实施计划中微调）
- `frontend/src/styles/main.css`、`pages.css` 等桌面断点样式
- 各 `views/*` 与相关展示组件的桌面布局分支
- `frontend/src/AppShell.spec.ts`、导航/组件单测、`frontend/e2e/responsive-shell.spec.ts` 等

### 7.2 Legacy `AppShell`

无 Vue Router 时的 `AppShell.vue` 保持可用；若改动成本低，可对齐分组标签文案，但不作为本规格主交付。

### 7.3 测试期望

- 单测：桌面侧栏分组、角色可见性、顶栏账户菜单与退出、当前路由标记。
- E2E：320/390/768 仍无横向滚动且移动底栏可见；1440 显示桌面导航、隐藏底栏；核心页面桌面下无溢出。
- 不要求本阶段新增视觉回归截图基线，除非实施中发现高风险样式分叉。

## 8. 分阶段交付建议

实施计划应拆为可独立验证的任务，建议顺序：

1. 桌面外壳：分组侧栏 + 顶栏 + 主内容结构；
2. 通用页面骨架与共享样式；
3. 管理页桌面表格/双栏；
4. 库存业务页（物品、库存、提醒、入库）；
5. 首页桌面化；
6. 测试补齐与回归。

## 9. 验收清单

- [ ] ≥1024px：侧栏分组、顶栏、主内容齐全，无移动底栏
- [ ] <1024px：底栏与现有移动路径不变
- [ ] ADMIN / MEMBER / VIEWER 导航可见性正确
- [ ] 全部已认证业务页与管理页具备桌面后台布局
- [ ] mint cream 视觉令牌未替换为深色/冷灰体系
- [ ] 相关 unit / e2e 通过

## 10. 开放实现细节

以下细节不阻塞规格批准，可在实施计划中做工程选择：

1. 首页是否完全移除大搜索，只保留顶栏搜索；
2. 物品详情用抽屉还是对话框；
3. 统计卡片是否抽成独立小组件；
4. 退出入口是否完全移除侧栏次要按钮。
